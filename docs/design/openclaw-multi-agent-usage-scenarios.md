## 一、OpenClaw 多 Agent 架构概览

### 1.1 当前配置示例（virtual-rnd-team）


### 1.2 Agent 角色与 Skill 定义

| Agent | 角色 | 核心能力 | 触发场景 |
|-------|------|----------|----------|
| **main (PM)** | 项目经理 | 编排虚拟团队、派发任务、质量门禁、自动重试 | 用户直接对话 |
| **analyst-agent** | 业务分析师 (BA) | 需求细化、竞品调研、PRD编写 | PM 收到模糊需求 |
| **architect-agent** | 系统架构师 (SA) | 系统设计、Mermaid图、代码评审(CR) | PRD审批后、代码完成后 |
| **devops-agent** | 开发+QA | 功能实现、自动化测试、部署 | 设计审批后、CR通过后 |

### 1.3 任务派发机制

```python
# PM 派发任务给子 Agent
sessions_spawn(
    agentId="analyst-agent",
    task="""
    [FEATURE_ID] user-auth
    [当前步骤] 第 1 项：需求细化
    [项目路径] /home/ubuntu/.openclaw/workspace-main/projects/my-app
    [产出路径] /home/ubuntu/.openclaw/workspace-main/projects/my-app/.staging/user-auth/
    [需求清单] /home/ubuntu/.../user-auth-checklist.md
    [CONTEXT FILES]
    - /home/ubuntu/.../requirements.md
    
    根据用户需求编写 PRD
    """,
    mode="run"  # 一次性任务
)
```

| 维度 | 类型 A | 类型 B | 类型 C |
|------|--------|--------|--------|
| **任务模式** | 串行流水线 | 并行独立任务 | 混合模式 |

## 七、总结

### 用户核心痛点

1. **看不到并行任务**：主 Agent 同时派发多个任务时，只显示一个
2. **不知道重试状态**：任务失败后是否在重试、用了什么策略
3. **看不到交互历史**：Agent 之间的请求/响应过程不透明
4. **分不清会话类型**：直接对话 vs 派发任务

### 推荐实施顺序

1. **P0 - Phase 1**：多任务并行展示（1-2天）
2. **P1**：任务生命周期与故障信息（3-5天）
3. **P2**：交互流可视化（可选，5-7天）

### 预期效果

完成后，Dashboard 将能够：
