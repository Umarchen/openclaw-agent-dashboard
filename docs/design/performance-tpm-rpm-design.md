> 文档版本：1.0
> 创建日期：2026-03-07
> 模块类型：性能监控核心模块

## 一、需求概述

### 1.1 功能目标

为用户提供实时的 API 调用量监控，帮助：
- 了解当前 Token/请求的消耗速率
- 查看历史趋势，识别高峰时段
- 为容量规划和成本控制提供数据支持

### 1.2 用户场景

| 角色 | 场景 | 目标 |
|------|------|------|
| 运维人员 | 日常监控 | 快速了解系统负载情况 |
| 成本管理者 | 费用分析 | 识别高消耗时段，优化资源分配 |
| 开发者 | 性能调优 | 发现异常调用模式 |

## 二、功能设计

### 2.1 时间范围选择

支持三种时间范围：

| 选项 | 数据点 | 刷新间隔 | 适用场景 |
|------|--------|----------|----------|
| **20分钟** | 20个点（每分钟1个） | 30秒 | 实时监控，快速定位问题 |
| **1小时** | 60个点（每分钟1个） | 1分钟 | 常规监控，趋势分析 |
| **24小时** | 24个点（每小时1个） | 5分钟 | 长期趋势，容量规划 |

**前端交互设计：**
- 使用 **分段按钮 (Segmented Control)** 切换时间范围
- 位置：指标卡片下方、趋势图上方
- 默认选中：20分钟

### 2.2 指标卡片

**保留指标：**

| 指标 | 计算方式 | 显示格式 | 说明 |
|------|----------|----------|------|
| **TPM** | 当前时间窗口的 Tokens/分钟 | `12,345` | 主指标，蓝色高亮 |
| **RPM** | 当前时间窗口的 Requests/分钟 | `123` | 辅助指标 |
| **总 Token** | 时间窗口内累计 Token | `1.2M` | 支持单位换算 |
| **总请求** | 时间窗口内累计请求 | `5,678` | - |

**移除指标：**

| 指标 | 原因 |
|------|------|
| ~~延迟~~ | 当前实现是 API 处理时间（~100ms），非 LLM 响应时间，无实际意义 |
| ~~错误率~~ | 后端未实现，始终为 0% |

**卡片布局（4列）：**

```
┌─────────────┬─────────────┬─────────────┬─────────────┐
│  TPM        │  RPM        │  总 Token   │  总请求     │
│  12,345     │  123        │  1.2M       │  5,678      │
│  ↑ 5%       │  ↓ 2%       │  本小时     │  本小时     │
└─────────────┴─────────────┴─────────────┴─────────────┘
```

### 2.3 趋势图

#### 2.3.1 图表类型

- **主图**：柱状图，显示 TPM
- **辅助**：折线图叠加 RPM（可选）

#### 2.3.2 交互设计

| 交互 | 行为 |
|------|------|
| **悬停** | 显示 Tooltip：时间、TPM、RPM |
| **点击柱体** | 弹出该分钟的调用详情面板 |
| **拖拽缩放** | 24h 模式支持拖拽选择时间段放大查看 |

#### 2.3.3 视觉设计

```
TPM 趋势
│
│     ▓▓▓  ▓▓▓
│  ▓▓▓▓▓▓▓▓▓▓▓▓  ▓▓▓
│──▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓──▓▓▓▓▓▓▓──
│
└──────────────────────────────────→ 时间
  13:00  13:05  13:10  13:15  13:20
```

- 柱体颜色：蓝色渐变（低→高）
- 高峰告警：超过阈值时柱体变为橙色/红色

### 2.4 统计摘要

在趋势图下方显示统计信息：

| 指标 | 20m/1h 显示 | 24h 显示 |
|------|-------------|----------|
| 平均 TPM | `Avg: 8,234` | `Avg: 5,678` |
| 峰值 TPM | `Peak: 15,678 @ 13:05` | `Peak: 25,000 @ 10:00` |
| 时间窗口 | `最近 20 分钟` | `最近 24 小时` |

## 三、后端设计

### 3.1 API 接口

**GET /api/performance**

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `range` | string | 否 | 时间范围：`20m`、`1h`、`24h`，默认 `20m` |

**响应结构：**

```json
{
  "current": {
    "tpm": 12345,
    "rpm": 123,
    "windowTotal": {
      "tokens": 1234567,
      "requests": 5678
    }
  },
  "history": {
    "tpm": [100, 200, 300, ...],
    "rpm": [10, 20, 30, ...],
    "timestamps": ["13:00", "13:01", "13:02", ...]
  },
  "statistics": {
    "avgTpm": 8234,
    "peakTpm": 15678,
    "peakTime": "13:05"
  }
}
```

### 3.2 数据聚合逻辑

#### 20m / 1h 模式

```
sessions/*.jsonl
    ↓ 过滤时间范围内的消息
    ↓ 按分钟分组
    ↓ 统计每分钟的 tokens/requests
    ↓
返回 20/60 个数据点
```

#### 24h 模式

```
sessions/*.jsonl
    ↓ 过滤最近24小时的消息
    ↓ 按小时分组
    ↓ 统计每小时的 tokens/requests
    ↓
返回 24 个数据点
```

### 3.3 修复项

| 问题 | 修复方案 |
|------|----------|
| 时间范围限制 1 小时 | 修改 `parse_session_file()` 支持 `range` 参数 |
| 硬编码 `~/.openclaw` | 使用 `OPENCLAW_HOME` 环境变量 |
| 错误率未实现 | **移除**该字段 |
| 延迟无意义 | **移除**该字段 |
